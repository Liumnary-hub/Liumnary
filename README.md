# 企业级进销存管理系统 API（后端）

> 基于 Spring Boot + Redis 的高性能进销存管理后台，解决高并发库存扣减、多维度统计查询、接口防刷、数据一致性等核心问题。

---

## 📌 项目简介

本项目是一个面向中小企业的进销存管理系统后端，提供商品管理、库存调度、员工权限、销售数据统计等核心功能。在高并发场景下，通过 Redis 分布式锁 + Lua 脚本实现精准库存扣减，利用二级缓存与事件机制将统计查询性能提升 45% 以上，并通过 AOP 实现无侵入式限流与操作日志记录。

**适用场景**：电商后台、仓储管理系统、零售门店管理。

---

## ✨ 核心技术亮点（面试/简历重点）

| 模块 | 技术方案 | 效果 |
|------|----------|------|
| **高并发库存扣减** | Redis 分布式锁 + Lua 原子脚本（锁粒度：商品+仓库） | 500 并发零超卖，TPS 提升 3 倍 |
| **多维度库存统计** | MySQL 聚合查询 + Redis 缓存 + Spring Event 主动失效 | 查询耗时 120ms → 65ms，提升 45% |
| **接口防刷限流** | AOP + Redis 固定窗口算法（支持 IP/用户/接口维度） | 登录接口 10 秒内最多尝试 3 次 |
| **操作日志记录** | AOP 环绕通知 + 异步保存 | 非侵入式记录，不影响主业务 |
| **无状态认证** | Spring Security + JWT | 支持分布式部署，无需 Session |
| **幂等性保证** | Redis SETNX 幂等令牌 | 防止网络重试导致重复扣减 |
| **数据最终一致性** | 每日定时对账 + 自动修复 + 告警 | 缓存与数据库差异自动修复 |
